# 硬件说明

## MCU

- 型号：NUC131SD2AE
- 内核：Arm Cortex-M0
- 目标核心频率：48 MHz
- 工程：Keil MDK 5 / ARM Compiler 5

## LCD

显示区域为 128×128，像素格式 RGB565，高字节先发送。屏幕接口没有 MISO，固件
只能确认写流程完成，不能读取控制器状态。

| 信号 | 引脚 | 电平和用途 |
| --- | --- | --- |
| SCL | PB15 | GPIO 模拟串行时钟 |
| SDA | PC14 | GPIO 模拟串行数据 |
| CS | PC15 | 低有效片选 |
| DC/RS | PC7 | 低=命令，高=数据 |
| RST | PA7 | 低有效硬复位 |
| LED | PC6 | 高有效背光 |

逻辑坐标需要叠加控制器偏移 `X+1`、`Y+2`。扫描方向寄存器为 `0x68`，像素格式
寄存器为 `0x05`。这些参数来自已验证硬件，不应在未重新上板验证时随意修改。

## GD25Q64

- 容量：8 MiB，地址 `0x000000`～`0x7FFFFF`
- JEDEC ID：`C8 40 17`
- 读取指令：`0x03` + 24 位地址
- SPI0：Mode 0、8 位、10 MHz

| 信号 | 引脚 |
| --- | --- |
| CS# | PC0 |
| SCLK | PC1 |
| SO/MISO | PC2 |
| SI/MOSI | PC3 |

当前固件只读取 GD25Q64，不提供在线擦除或编程接口。资源镜像需要使用外部编程器写入。

## 播放调度

TIMER0 每秒产生 30 次帧事件，中断只累加事件计数。Flash 读取、RLE 解码和 LCD
刷新均在主循环执行。事件累计超过 4 次进入青屏错误，不通过丢帧改变原始动画速度。
