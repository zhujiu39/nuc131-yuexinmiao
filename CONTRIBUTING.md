# 贡献指南

感谢参与改进本项目。

## 提交前

1. 从新分支完成修改，不提交 Keil `obj/`、`lst/` 和用户界面状态文件。
2. 不提交没有公开或再分发权利的原始视频、图片、字体和音频。
3. 修改资源格式时，同步更新固件解析器、生成器、校验器和格式文档。
4. 修改 C 代码后使用 Keil 强制全量编译，并记录最终错误和警告数量。
5. 修改资源后运行 `validate_animation_pack.py`，确保结果为 `PASS`。
6. 运行 `git diff --check`，避免空白字符和合并噪声。

## 代码要求

- 固件保持裸机主循环结构，中断只记录短事件。
- 外设等待必须有超时，所有地址和长度必须检查边界。
- 默认不使用动态内存。
- 固件 C 注释使用简体中文结构化格式，并说明硬件和并发约束。
- Python 工具保持标准库优先，异常必须返回非零退出码。

## Pull Request

PR 请写清硬件型号、修改原因、验证方法、构建结果以及是否改变资源二进制。
